# 读书分享文案生成器

从 EPUB/TXT/Markdown 书籍中提取核心内容和推荐价值，搜索外部书评和阅后感作为补充素材，生成有感染力、有观点的剪映图文成片读书分享文案，并通过本地 SQLite 数据库（`.claude/book-share-tracker.db`）追踪已处理书籍，避免重复生成。

## 核心理念

**书籍是故事，分享是桥梁。** 不做机械的"这本书讲了什么"——而是找到这本书最打动人的那个点，用有感染力的方式告诉观众：**"这本书值得你花时间去读。"**

好的读书分享文案的标准：看完让人觉得 **"我想读这本书"** 或 **"我也经历过这个"** 或 **"原来是这样"** ——不是被塞了一个书单，而是被点燃了好奇心、被戳中了内心。

## 使用方法

```
/book-share <书籍路径> [--style 风格] [--output 输出目录] [--notion] [--notion-parent PAGE_ID]
```

- `<书籍路径>`：**必填**。EPUB/TXT/Markdown 格式的书籍文件路径
- `--style`：指定视频风格（深度解读/情感共鸣/快速种草/金句分享/冷门推荐），默认自动匹配最佳风格
- `--output`：指定输出目录，默认为 `studio/`
- `--notion`：上传文案到 Notion（默认不上传，仅生成本地文件）
- `--notion-parent {page_id}`：指定 Notion 父页面 ID，默认读取环境变量 `NOTION_PARENT_PAGE_ID`

示例：
- `/book-share downloads/小王子.epub`
- `/book-share downloads/被讨厌的勇气.epub --style 深度解读`
- `/book-share ~/Desktop/活着.epub --output studio/`
- `/book-share downloads/三体.epub --style 快速种草 --notion`

## 数据库工具

所有 SQLite 操作通过 `.claude/book_share_schema.py` 提供的 Python 函数完成。技能执行过程中，以 `python3 -c` 或 heredoc 方式调用这些函数，无需直接写 SQL。

关键函数：
- `add_book(title, author, source_path, book_type, word_count, date) → book_id`
- `get_processed_books() → list` — 获取所有已处理书籍
- `is_book_processed(title, author) → bool` — 检查是否已处理
- `get_book_scripts(book_id) → list` — 获取某本书已有脚本
- `add_script(book_id, style, title, path, date) → script_id`
- `update_notion_upload(book_id, style, page_id, page_url)`
- `get_stats() → dict` — 全局统计
- `get_all_books_with_scripts() → list` — 所有书籍及脚本

调用方式示例：
```bash
python3 -c "
import sys; sys.path.insert(0, '.claude')
from book_share_schema import *
init_db()
stats = get_stats()
print(stats)
"
```

## 核心约束

1. **外部搜索必做**：与 healing-video/science-video 不同，book-share 的差异化核心就是外部书评搜索。**不可跳过**。必须通过搜索获取豆瓣评分、读者评价、公众号书评文章等信息。
2. **每处理一本书，必须立即通过 `book_share_schema.py` 写入 SQLite 数据库**。这是防止重复处理的底线机制。
3. **推荐价值 > 内容复述**：文案的核心不是"这本书讲了什么"，而是"为什么你要读这本书"。

## 工作流：四阶段（+ 可选 Notion 上传）

### 阶段一：书籍发现（Discovery）

**目标**：解析书籍文件，提取元数据和核心内容，了解这本书"是什么"。

**步骤**：

1. **解析参数**：从 `$ARGUMENTS` 中提取书籍路径、`--style`、`--output`、`--notion`、`--notion-parent`

2. **判断文件类型并提取内容**：
   - `.epub` → 用 Python `zipfile` 解压（注意 UTF-8 编码修复），读取 `nav.xhtml` 获取目录结构，读取 `content.opf` 获取元数据
   - `.txt` → 直接读取，按章节标题拆分
   - `.md` → 直接读取，按 heading 拆分

3. **EPUB 提取参考代码**（在 Bash 中以 `python3 << 'PYEOF'` 方式执行）：
```python
import zipfile, os, shutil, re
from html.parser import HTMLParser

class TextExtractor(HTMLParser):
    def __init__(self):
        super().__init__()
        self.texts = []
        self.skip = False
    def handle_starttag(self, tag, attrs):
        if tag in ('script', 'style'): self.skip = True
    def handle_endtag(self, tag):
        if tag in ('script', 'style'): self.skip = False
    def handle_data(self, data):
        if not self.skip:
            t = data.strip()
            if t: self.texts.append(t)

def read_epub_structure(epub_path):
    """解压并读取 EPUB 结构和内容"""
    import tempfile
    tmp = tempfile.mkdtemp()
    with zipfile.ZipFile(epub_path, 'r') as zf:
        for info in zf.infolist():
            name = info.filename
            try: name = name.encode('cp437').decode('utf-8')
            except: pass
            target = os.path.join(tmp, name)
            os.makedirs(os.path.dirname(target), exist_ok=True)
            if not info.is_dir():
                with zf.open(info) as s, open(target, 'wb') as d:
                    d.write(s.read())
    return tmp

def extract_text_from_xhtml(filepath):
    """从 XHTML 提取纯文本"""
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()
    extractor = TextExtractor()
    extractor.feed(content)
    return '\n'.join(extractor.texts)
```

4. **读取书籍核心内容**——至少阅读前言/序言、目录、第一章和最后一章，形成对书籍的基本理解：
   - 这本书的核心主题是什么？
   - 作者想传达什么核心观点或情感？
   - 书籍的大致结构（章节数、主题分布）
   - 总字数估计

5. **识别书籍类型**——根据内容自动判断：
   - `小说/文学`：虚构叙事，有情节和人物
   - `思想/哲学`：探讨抽象概念，有思想深度
   - `经管/商业`：实用方法论，案例驱动
   - `心理/成长`：关注内心世界，自我提升
   - `历史/社科`：基于事实和研究的叙事
   - `科普/知识`：传播科学知识
   - `传记/纪实`：真实人物或事件记录
   - `生活/治愈`：轻松温暖，关注日常

6. **检查是否已处理**——调用 `is_book_processed(title, author)` 检查。如果已处理，列出已有脚本的风格，询问用户是否要生成新风格。

7. **向用户展示书籍概况**：
   ```
   📖 《书名》
   ✍️  作者：{作者}
   📂 类型：{书籍类型}
   📄 篇幅：约 {N} 字 / {N} 章
   💡 核心主题：{一句话描述}
   ```

### 阶段二：调研与丰富（Research）⭐ 核心差异化

**目标**：搜索外部书评、阅后感、讨论，找到这本书最打动人的角度和素材。**这是 book-share 与其他命令最大的区别——不做外部调研，就无法生成有说服力的推荐文案。**

**⚠️ 搜索降级策略（严格执行，不可跳过）**：

**优先级 1**：使用 WebSearch 工具进行搜索
**优先级 2**：WebSearch 返回错误后，降级使用 Tavily Search API：
```bash
curl -s -X POST https://api.tavily.com/search \
  -H "Content-Type: application/json" \
  -d '{"api_key":"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","query":"搜索内容","max_results":5}'
```
**优先级 3**：Tavily 也失败，使用 Playwright MCP 抓取网页
**优先级 4**：所有搜索工具都失败，在文案 frontmatter 中标注 `unverified`

#### 2.1 第一轮搜索：了解这本书的公众认知

**通用搜索（WebSearch / Tavily）**：
- `{书名} 书评` — 了解整体评价
- `{书名} 豆瓣` — 获取评分和代表性短评
- `{书名} 读后感` — 了解读者真实感受

**微信搜一搜**（`mcp__cimi-data__search_articles_wechat`，**必执行**）：
- `{书名}` — 搜索公众号深度书评文章
- `{书名} 推荐` — 搜索推荐类文章

**目标产出**：
- 豆瓣评分（如有）
- 3-5 条最有代表性的读者评价（正反面都要）
- 这本书被推荐的理由中最常出现的 3 个关键词
- 公众号深度书评文章 2-3 篇

#### 2.2 第二轮搜索：寻找差异化角度

**通用搜索**：
- `{书名} 最打动人的地方` — 寻找情感触点
- `{书名} 为什么值得读` — 寻找推荐论点
- `{书名} 适合什么人读` — 寻找目标受众画像
- `{书名} 冷知识` 或 `{书名} 背后的故事` — 寻找独特冷门素材

**微信搜一搜（必执行）**：
- `{书名} 深度` 或 `{书名} 解读` — 寻找有深度的分析视角
- `{书名} 最精彩的` 或 `{书名} 金句` — 寻找书中精华

**获取全文**：搜索到微信文章 URL 后，使用 `mcp__cimi-data__get_article_body` 获取文章正文内容。

**目标产出**：
- 这本书最打动人的 1 个核心角度
- 3-5 句可以作为金句的原文
- 1-2 个关于作者或创作背景的独特素材
- 与同类书籍的差异化定位

#### 2.3 第三轮搜索（可选）：深度挖掘

- `{书名} 作者 创作背景` — 了解创作动机和故事
- `{书名} 社会影响` 或 `{书名} 为什么火` — 了解社会影响力
- `{书名} 类似的书` — 为差异化对比找素材

#### 2.4 搜索结果存档

**⚠️ 搜索结果必须存档**。在输出目录下创建 `attachments/` 目录，保存搜索索引和文章内容。

**搜索索引文件**（`attachments/搜索索引.md`）记录所有搜索关键词、搜索工具、结果数、关键发现。

**文章内容文件**（`attachments/文章{N}-{标题}.md`）保存搜索到的每篇文章全文，供后续事实核查和文案生成回溯引用。

### 阶段三：文案生成（Generation）

**目标**：基于书籍内容 + 外部调研素材，生成剪映图文成片读书分享文案。

#### 3.1 自动匹配最佳风格

如果用户未指定 `--style`，根据书籍类型和调研发现自动选择最佳风格：

| 书籍类型 | 默认风格 | 理由 |
|----------|----------|------|
| 思想/哲学/社科 | 深度解读 | 有思想深度，适合深入剖析 |
| 小说/文学 | 情感共鸣 | 有人物命运，适合情感切入 |
| 经管/商业/工具书 | 快速种草 | 有实用价值，适合痛点驱动 |
| 心理/成长/传记 | 情感共鸣 | 有成长弧光，适合励志角度 |
| 历史/科普 | 深度解读 | 有知识含量，适合反常识切入 |
| 冷门/小众书籍 | 冷门推荐 | 天然差异化，好奇心驱动 |
| 诗歌/散文/语录 | 金句分享 | 语言本身就是最好的推荐 |

**风格选择指南**（如果调研发现更强角度，可覆盖默认匹配）：
- 外部书评中情感共鸣评价特别多 → 优先情感共鸣
- 书中有大量优质金句 → 优先金句分享
- 书籍核心观点极其反常识 → 优先深度解读
- 书籍实用性极强 → 优先快速种草

#### 3.2 读书分享文案创作技巧

##### 五种风格指南

**风格一：深度解读**
- **开场钩子**：一个颠覆大众认知的观点或反常识的结论
- **中间推进**：层层深入——先讲大众的误解 → 再讲书中揭示的真相 → 最后讲这个真相如何改变你看世界的方式
- **结尾**：一个让人想立刻翻书验证的悬念
- **适合书籍**：《人类简史》《思考快与慢》《乌合之众》

**风格二：情感共鸣**
- **开场钩子**：一个让人心酸或温暖的书中片段，用第一人称重述
- **中间推进**：先讲书中人物的困境（让观众代入）→ 再讲转折或救赎（情绪升温）→ 最后讲这个故事的普世意义
- **结尾**：一个温暖而有力的句号
- **适合书籍**：《活着》《小王子》《你当像鸟飞往你的山》

**风格三：快速种草**
- **开场钩子**：直击痛点——一句话让观众觉得"这就是我"
- **中间推进**：痛点放大 → 这本书怎么解决（给方法不给鸡汤）→ 用了之后的变化
- **结尾**：一个具体可操作的行动号召
- **适合书籍**：《原子习惯》《非暴力沟通》《搞定》

**风格四：金句分享**
- **开场钩子**：直接抛出书中最炸的一句金句
- **中间推进**：第一句金句 + 背后故事 → 第二句金句 + 为什么戳中你 → 第三句金句 + 放在一起的力量
- **结尾**：用一句原文收尾，留白
- **适合书籍**：《人生海海》《瓦尔登湖》《沉思录》

**风格五：冷门推荐**
- **开场钩子**："冷门但炸裂"的强烈反差
- **中间推进**：为什么冷门 → 为什么炸裂（具体细节证明）→ 读完的感受
- **结尾**：制造稀缺感和社群感
- **适合书籍**：评分高但标记人数少的冷门好书

##### 通用写作规范

0. **⚠️ 内容安全红线（强制遵守，违反即重写）**：
   - ❌ 禁止对品牌/公司进行负面批评
   - ❌ 禁止对政府/政策进行批判
   - ❌ 禁止制造群体对立
   - ❌ 禁止过度负面情绪宣泄
   - ✅ 替代方案：用"这本书让我意识到..."替代批判；用"我们可以..."替代"XX应该..."

1. **情绪线不能断（最重要，优先级高于一切）**：
   - 抖音观众划走只有一个原因：情绪断了。情绪一断，一切白费
   - **每种风格都有不同的情绪曲线（见上方各风格描述）**
   - **⚠️ 反塌房铁律**：视频后半段禁止变成以下任何一种——
     - ❌ 书单罗列（"除此之外还有 XX、YY、ZZ 也值得读"）
     - ❌ 作者生平报告（"作者 1968 年出生于..."）
     - ❌ 教科书式总结（"这本书一共分为三部分..."）
     - ❌ 读书方法论说教（"我们应该这样读书..."）
     - ❌ 强行升华（"所以我们要珍惜当下，活在当下"）
   - **后半段该怎么写**：用一个更大的反转、一个更深的触动、一句更炸的金句，或者一个让观众忍不住翻书的悬念来收尾

2. **推荐价值是核心**：观众看完不需要知道这本书的全部内容——只需要知道"这本书能给我什么"

3. **结构适配剪映分镜**：
   - 用空行分段，每段对应一个镜头画面
   - 每段 1-2 句话，15-40 字
   - 总字数 600-750 字（对应 2-3 分钟视频，按语速 250 字/分钟）
   - **段落数 10-12 段**，每一帧都要有存在的理由
   - **⚠️ 无填充段规则**：删掉该段后视频仍然通顺 = 这段是废话，必须删
   - 首段至少 15 字以上

4. **开头 3 秒钩子**：用反常识/悬念/数据炸弹/情感冲击在 3 秒内抓住注意力，绝不能是"今天推荐一本书"

5. **口语化表达**：短句为主，像一个爱读书的朋友在跟你聊天，不是朗读课文

6. **具象化改写（剪映适配核心）**：

| 类型 | 问题 | 改写方向 |
|------|------|---------|
| 抽象概念（"自由""孤独""成长"） | 剪映无法匹配画面 | 替换为具体的人/物/场景/行为 |
| 心理活动（"他突然明白了"） | 无法拍摄 | 替换为外化的动作/表情/场景变化 |
| 书中原句（过于文学化） | 听读体验差 | 改成口语，保留原意，不保留原文 |
| 数据/研究结论 | 需要视觉锚点 | 加具体场景，如"相当于你每天..." |

7. **结尾**：用书中一个细思极恐的细节/一个让人无法平静的金句留白——不能是"关注我""你怎么看评论区告诉我"

8. **反cliché规则（严禁使用）**：
   - ❌ "今天要给大家推荐一本好书" / "这本书改变了我的人生" / "读完我才明白"
   - ❌ "毫不夸张地说" / "强烈推荐" / "相信我" / "你怎么看评论区告诉我"
   - ✅ 替代方案：用一个具体的场景/细节/感受来推荐，让观众自己得出结论

##### 差异化策略（至少用一种，优先组合使用）

- **反直觉切入**：从大众对这本书的误解入手，用书中的真相打破误解
- **普通人关联**：不讲书的理论，讲这本书如何改变了一个普通人的具体生活
- **隐藏的赢家/输家**：书中最不起眼的角色/细节，却藏着最深的真相
- **跨时空对比**：这本书的某个观点，在 XX 年后的今天应验了/被颠覆了
- **第一人称叙事**：用"我读这本书的时候..."的视角，而不是上帝视角介绍书
- **放大最毒的细节**：书中最触动你的那一个细节，放大为全片核心线索

##### 素材搜索列规范

**⚠️ 强制规则：每个画面的素材搜索方案必须包含【YouTube】和【Pexels】两个来源标签。不允许只有【Pexels】没有【YouTube】的情况。**

| 来源 | 适用场景 | 搜索词构建要点 |
|------|---------|---------------|
| **YouTube** | **首选来源**。书封展示、翻书实拍、作者采访、读书会视频、书店场景 | **必须包含书名/作者名具体关键词**，加 `book review` `reading` `书评` `采访` 等后缀 |
| **Pexels** | 氛围画面（阅读场景、书架、咖啡、自然等通用场景）、情感空镜头、转场过渡 | 用英文搜索；2-3 个词组合，如 `reading+book+cozy` |
| **表情包** | 吐槽段、反转段、情绪转折段、纯观点段 | 格式：`【表情包】情感词+场景词`，如 `震惊+张大嘴` `感动+泪目` |

**搜索词构建规则**：
1. **书籍强相关（最重要）**：搜索词必须包含书名/作者名等具体关键词，严禁泛词
   - 好：`小王子+圣埃克苏佩里+书封` → 差：`book+cover`
2. **每个画面提供至少 3 套搜索词**（每个来源标签下）：精准定位 + 补充角度 + 宽泛兜底
3. **中英双语**：YouTube 用中文（国内读书博主视频为主），Pexels 用英文
4. **搜索词总数**：每个画面至少 6 个搜索词组（YouTube 3个 + Pexels 3个）
5. **避免无效搜索词**：❌ `data+chart` `speaker+closeup` → ✅ `bookstore+shelf+books` `girl+reading+sunlight`

#### 3.4 文案自检（12 项质量检查，强制执行）

**⚠️ 这是一个强制执行的质量关卡，不可跳过。每个文案文件必须逐项通过以下 12 条检查。**

| # | 检查项 | 检查方法 | 不通过则 |
|---|--------|----------|----------|
| 0 | **⚠️ 内容安全** | 是否存在对品牌/公司的负面批评？对政府/政策的批判？制造群体对立？过度负面情绪？ | 删掉或用正向角度重写 |
| 1 | **素材搜索格式** | 每行素材搜索是否同时包含【YouTube】和【Pexels】两个标签？每个标签下是否≥3套搜索词？ | 补充缺失标签和搜索词 |
| 2 | **段落数** | 台词纯文本段落数是否在 10-12 段范围内？ | 多则合并相似场景，少则拆分过长的段落 |
| 3 | **总字数** | 台词纯文本总字数是否在 600-750 字？ | 超出则删减填充段，不足则补充关键细节 |
| 4 | **反塌房** | 后半段（第 7 段起）是否存在书单罗列/作者生平报告/教科书总结/读书方法论说教/强行升华？ | 删掉，用更大的反转、更深的触动或更炸的金句替换 |
| 5 | **反cliché** | 是否包含禁用话术："今天要给大家推荐" / "这本书改变了我的人生" / "读完我才明白" / "毫不夸张地说" / "强烈推荐" / "相信我" / "你怎么看评论区告诉我"？ | 替换为具体场景/细节/感受 |
| 6 | **正向情绪曲线** | 逐段读台词，情绪是否持续升级？升级方向是否正向（被点燃/被治愈/被触动/被启发）？是否有断点或转向负面？ | 删掉断点段或负面情绪段 |
| 7 | **3 秒钩子** | 第 1 段台词是否能在 3 秒内抓住注意力？是否是"今天推荐XX"式的无效开场？ | 重写第 1 段 |
| 8 | **差异化** | 这个推荐角度跟其他读书博主的推荐角度是否不同？搜索确认是否已有大量同类内容？ | 换角度重写或放弃 |
| 9 | **事实核查** | frontmatter 中 verified_claims 和 unverified_claims 是否完整？具体数据是否已验证或改为模糊表述？ | 补充核查结果 |
| 10 | **具象化** | 是否存在抽象概念/隐喻/文学化表达未被替换为具体场景？ | 替换为具体的人/物/场景 |
| 11 | **推荐价值验证** | 读完台词全文，观众看完后最可能的反应是什么？"我想读这本书" / "原来是这样" / "我也经历过"？如果观众反应是"跟我有什么关系"或"听完但是没记住"，必须重写。 | 强化推荐角度，放大情感触点 |

## 错误处理

| 场景 | 处理 |
|------|------|
| 书籍文件不存在 | 提示检查文件路径 |
| 文件格式不支持 | 提示支持 EPUB/TXT/Markdown |
| EPUB 解析失败 | 降级方案：手动解压 + 逐个 XHTML 读取 |
| 书籍已处理同风格 | 提示已有同风格脚本，询问是否覆盖 |
| WebSearch 不可用 | **⚠️ 必须降级到 Tavily API（curl 调用），不可跳过** |
| 所有搜索工具都失败 | 基于书籍内容直接生成，标注"外部搜索不可用" |
| 搜索无书评结果 | 基于书籍内容生成，标注"无外部书评参考" |
| 事实核查无法验证 | 记录到 unverified_claims，具体数据模糊化 |
| 后半段塌房 | **⚠️ 删掉报告/说教，用反转/金句/触动替换** |
| 内容安全违规 | **⚠️ 必须重写，用正向角度切入** |
| Notion 上传失败 | 本地文件不受影响，记录失败原因 |

## 数据库表结构

SQLite 数据库 `.claude/book-share-tracker.db`，通过 `.claude/book_share_schema.py` 操作。

```
books
├── id (PK)
├── title             — 书名
├── author            — 作者
├── source_path       — 书籍文件路径
├── book_type         — 书籍类型
├── word_count        — 字数估计
├── processed_date    — YYYY-MM-DD
└── UNIQUE(title, author)

scripts
├── id (PK)
├── book_id (FK → books.id)
├── style             — 深度解读/情感共鸣/快速种草/金句分享/冷门推荐
├── title             — 视频标题
├── path              — 文案文件路径
├── generated_date    — YYYY-MM-DD
├── notion_page_id    — Notion 页面 ID（可空）
├── notion_page_url   — Notion 页面 URL（可空）
├── notion_uploaded   — 0 或 1
└── UNIQUE(book_id, style)

视图：v_stats（全局统计）、v_book_scripts（书籍脚本列表）
```

**操作规范**：
- 始终通过 `book_share_schema.py` 的函数操作数据库，不手写 SQL
- 每次 `python3 -c` 调用前先 `sys.path.insert(0, '.claude')`

## 其他要求

- **所有面向用户的输出用中文**
- **推荐内容不得捏造**——所有观点和引用必须来自书籍原文或经过搜索验证的外部书评
- **段落间用空行分隔**——台词纯文本每段之间留空行
- **每个阶段结束必须报告进度**——让用户知道搜索了多少篇书评、找到了什么关键信息
- **输出文件命名**：`{output_dir}/{YYYY-MM-DD}/{HHMM}/选题1-{标题缩写}.md`
- **视频标题自拟**——标题本身就应该有观点有态度，不要用"XX读书分享"这种泛标题
