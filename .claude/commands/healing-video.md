# 鸡汤文案生成器

从 EPUB/TXT/Markdown 书籍中提取文章，提炼核心观点和人生洞察，生成有力量、有态度的鸡汤文案（同时输出长文版和金句版），并通过本地 SQLite 数据库（`.claude/healing-tracker.db`）追踪已处理故事，避免重复阅读。

## 核心理念

**书籍是矿，观点是金。** 不复述故事，不搬原文——从书里提炼出能让人"被点燃、被治愈、想转发"的观点，用有力量的方式重新表达。

好鸡汤文案的标准：看完让人觉得**"对！就是这样"**或**"我也要这样"**——不是被灌了糖水，而是被理解了、被点醒了、被推了一把。

## 输出格式

每次生成同时输出两个版本：
1. **长文版**（200-500字）：完整鸡汤文，有开头钩子、论述推进、金句结尾
2. **金句版**（3-10句）：每句能独立截图转发，情绪递进

## 使用方法

```
/healing-video <书籍路径> [--output <输出目录>]
```

- `--output`：可选，指定文案输出目录。默认为 `.outputs/`。目录不存在时会自动创建。
- 支持相对路径（如 `--output outputs/reader`）和绝对路径（如 `--output ~/Desktop/healing`）。

示例：
- `/healing-video downloads/读者（2021年第1期）.epub`
- `/healing-video downloads/读者（2021年第1期）.epub --output outputs/reader`
- `/healing-video ~/Desktop/某本书.txt --output ~/Desktop/鸡汤文案`

## 数据库工具

所有 SQLite 操作通过 `.claude/healing_schema.py` 提供的 Python 函数完成。技能执行过程中，以 `python3 -c` 或 heredoc 方式调用这些函数，无需直接写 SQL。

关键函数：
- `add_book(name, source_path, source_type, issue, date) → book_id`
- `get_processed_titles(book_name) → set[str]` — 获取已处理标题集合
- `add_story(book_id, title, author, section, word_count) → story_id`
- `update_evaluation(story_id, score, tags, best_styles, notes)`
- `mark_selected(story_id)` / `mark_rejected(story_id)`
- `add_script(story_id, style, path, date)`
- `get_stats() → dict` — 全局统计
- `get_book_progress(book_name?) → list` — 书籍处理进度

调用方式示例：
```bash
python3 -c "
import sys; sys.path.insert(0, '.claude')
from healing_schema import *
init_db()
titles = get_processed_titles('读者（2021年第1期）')
print(titles)
"
## 核心约束

**最重要**：每处理一个故事，必须立即通过 `healing_schema.py` 写入 SQLite 数据库。无论该故事被选中生成脚本还是被筛掉，都要标记为"已处理"。这是防止重复阅读的底线机制。数据库文件位于 `.claude/healing-tracker.db`。

## 工作流：四阶段

### 阶段一：发现（Discovery）

**目标**：提取书籍元数据和所有可读文章/章节列表，过滤已处理项。

**步骤**：

1. **判断文件类型**：
   - `.epub` → 用 Python `zipfile` 解压（注意 UTF-8 编码修复），读取 `nav.xhtml` 获取目录，读取 `content.opf` 获取元数据
   - `.txt` → 直接读取，按章节标题拆分
   - `.md` → 直接读取，按 heading 拆分

2. **EPUB 提取参考代码**（在 Bash 中以 `python3 << 'PYEOF'` 方式执行）：
```python
import zipfile, os, shutil, re
from html.parser import HTMLParser

class TextExtractor(HTMLParser):
    def __init__(self):
        super().__init__()
        self.texts = []
        self.skip = False
    def handle_starttag(self, tag, attrs):
        if tag in ('script', 'style'): self.skip = True
    def handle_endtag(self, tag):
        if tag in ('script', 'style'): self.skip = False
    def handle_data(self, data):
        if not self.skip:
            t = data.strip()
            if t: self.texts.append(t)

def read_epub_structure(epub_path):
    """解压并读取 EPUB 结构和内容"""
    import tempfile
    tmp = tempfile.mkdtemp()
    with zipfile.ZipFile(epub_path, 'r') as zf:
        for info in zf.infolist():
            name = info.filename
            try: name = name.encode('cp437').decode('utf-8')
            except: pass
            target = os.path.join(tmp, name)
            os.makedirs(os.path.dirname(target), exist_ok=True)
            if not info.is_dir():
                with zf.open(info) as s, open(target, 'wb') as d:
                    d.write(s.read())
    return tmp

def extract_text_from_xhtml(filepath):
    """从 XHTML 提取纯文本"""
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()
    extractor = TextExtractor()
    extractor.feed(content)
    return '\n'.join(extractor.texts)
```

3. **查询已处理故事**——通过 `get_processed_titles(book_name)` 获取该书已处理标题集合，过滤后得到待处理列表。

4. **注册书籍**——调用 `add_book(name, source_path, source_type, issue, date)` 获取 `book_id`（已存在则返回已有 id）。

5. **列出待处理故事**：输出编号、标题、作者、栏目（如有）、字数估计。每页显示 20 个。

6. **如果没有新故事**（全部已处理），告知用户并结束。

### 阶段二：评估（Evaluation）

**目标**：逐批评估文章的观点提炼价值和鸡汤转化潜力，对每个故事输出评分。

**评估维度**（每项 1-10 分，不显示给用户，仅作为思考依据）：

| 维度 | 说明 | 高分信号 | 低分信号 |
|---|---|---|---|
| **观点提炼价值** ⭐ | 书中是否有能提炼成独立人生道理的内容 | 有明确的观点、态度、人生洞察 | 纯叙事无观点、纯科普、流水账 |
| **鸡汤转化潜力** | 观点能否写成让人"被点燃/被治愈/想转发"的文案 | 有力量感、有共鸣、不空洞 | 观点太窄、太小众、太空泛 |
| **金句密度** | 有没有能直接作为社交媒体文案的原句或改编句 | 短小精悍、有节奏、有态度 | 大白话流水账、官腔、说教腔 |
| **共鸣广度** | 观点能否击中大众的日常感受 | 关于成长、放手、自律、心态、人际关系等 | 过于小众、专业性太强、与普通人无关 |

> **观点提炼价值是第一筛选标准**：不是每篇文章都有值得提炼的观点。纯叙事、纯科普、纯数据、让人绝望无力的——直接淘汰。我们需要的是能让人停下来想一想、然后觉得"有道理"的内容。

**输出格式**——评估汇报以表格呈现：

```
| # | 标题 | 作者 | 栏目 | 字数 | 评分 | 核心观点候选 | 标签 |
|----|------|------|------|------|------|-------------|------|
| 1 | 夜晚疲惫至极 | 帕慕克 | 文苑 | ~300 | ⭐⭐⭐⭐⭐ | 允许一切发生、独处的力量 | #独处 #成长 |
| 2 | 电子废弃物 | — | 点滴 | ~200 | ⭐ | ❌ 无可提炼观点 | #科普 |
```

评分用星级：⭐⭐⭐⭐⭐(9-10)  ⭐⭐⭐⭐(7-8)  ⭐⭐⭐(5-6)  ⭐⭐(3-4)  ⭐(1-2)

评估完成后，**立即写入数据库**——对每个评估过的故事调用 `add_story()` + `update_evaluation()`，状态为 `evaluated`。

询问用户：哪些文章要生成文案？可以选编号（如 `1,5,7-9`），也可以说"全部高分的"或"评分 4 星以上的"。

### 鸡汤文案创作技巧（必须在生成前内化）

文案的核心不是"复述故事"，而是**提炼观点，用有力量的方式表达出来**。书籍是原材料，文案是从中炼出的精华。

#### 开头钩子：第一句话决定生死

开头必须让人**手指悬停、不会划走**。两种模式，根据文章内容选择：

**模式A：观点直给型**——第一句就亮出态度
> 人只要铆足劲往前走，就一定会有新的东西在前面。
> 人生最顶级的心态就是允许一切发生。
> 不是赢了才叫厉害，能主动退出消耗自己的局才厉害。

写法：用**绝对化句式**（一定、就、就是、才叫），第一句就是结论。观众要么认同、要么想反驳——不管哪种，都不会划走。

**模式B：反差冲击型**——用对比制造认知碰撞
> 所有人都在告诉她，你见识到社会的厉害了吧。他说不，我见识到我自己的厉害了。
> 别人拼命合群，她选择独来独往。结果她活得比谁都好。

写法：**"别人/所有人…，他/她…不/却…"**结构。前半句是大众认知，后半句是反常识的真实答案。越短越有力。

#### 中间推进：让观众觉得"对对对就是这样"

钩子之后需要展开论述。三种常用写法（可混搭）：

**1. 递进式**——一层比一层深入
> 你得先学会拒绝。拒绝那些消耗你的人。然后你才有精力，去靠近真正值得的事。
写法：**先…然后…最后…**，逻辑清晰，像在给观众一套方法。

**2. 举例式**——用一个极短的场景支撑观点
> 真正厉害的人，从不解释自己。就像那棵长在悬崖边的树，没人问它为什么长在那里，它就是长成了最挺拔的样子。
写法：一个观点 + 一个具体场景/比喻。例子要短（1-2句），不能变成讲故事。

**3. 对比式**——用"不是A，是B"强化观点
> 成熟不是变得冷漠，是变得温柔但有边界。强大不是不害怕，是害怕了依然往前走。
写法：**"不是…，是…"** 是鸡汤文最强句式之一。前半句打破误区，后半句给出真相。

> **关键**：如果开头用了"观点直给型"（如"谁影响我，我就放弃谁"），中间要继续写**怎么做**——给方法、给步骤、给行动指南，不能只甩一个观点就结束。

#### 结尾技巧：必须让人被点燃/被治愈/想转发

结尾是观众决定要不要截图转发的时刻。结尾不能是总结、不能是感叹、不能是叹息——必须是一个**有力量的句号**。

**模式1：观点定调型**——用一句提炼过的人生道理扎住全文
> 人只要铆足劲往前走，就一定会有新的东西在前面。
> 你今天的坚持，是你多年前祈求的模样。
> 真正的自由，不是想做什么就做什么，而是敢不做自己不想做的事。

写法：**陈述句，用绝对词**（一定、就、就是、才是）。语气笃定，不留余地。

**模式2：反差回击型**——用对比制造反转，让人"啪"一下被打醒
> 所有人都在告诉她，你见识到社会的厉害了吧。他说不，我见识到我自己的厉害了。
> 他们说你输了。但只有你知道，你赢了自己。

写法：**"别人说/所有人…，但…/不…"**。越短越有力——目标是让观众想拍大腿。

**模式3：行动指令型**——直接告诉观众"你该怎么做"
> 谁影响我，我就放弃谁。
> 从今天开始，只做让自己增值的事。
> 别等了，现在就是最好的时候。

写法：**祈使句或"谁…就…"句式**。不是建议、不是商量——是像朋友拽着你说"别犹豫了，就这样干"。

#### 金句版创作规则

金句版是长文版的精华提炼，每句必须能独立截图转发：
- **3-10句**，情绪递进（从轻到重，或从共鸣到点燃）
- **每句15-35字**，不能太短（太空洞）也不能太长（截图不好看）
- **三种句式混搭**：观点陈述（"…就是…"）、反差对比（"不是…是…"）、行动指令（"…就…"）
- **不用连接词**——每句独立成段，不需要"所以""因此""也就是说"
- **不要"鸡汤味"太重**——避免"加油""努力""你最棒"这种空洞鼓励。有力量的鸡汤让人觉得自己被理解了，不是被灌了糖水

#### 文案风格底线

- **说人话**：像朋友圈里一个活得通透的朋友在说话，不是播音腔、不是文艺腔
- **有态度**：鸡汤文必须有立场——支持什么、反对什么、倡导什么。不能两端都讨好
- **有节奏**：短句为主，每句15-25字。长句拆短，阅读体验更好
- **不编造**：所有观点必须来自书籍原文，可以提炼、改编、重组，但不能凭空捏造

**自查清单**（生成每篇文案后必问）：
- 开头第一句能让人停下手指吗？→ 不能就重写
- 观点是从书里提炼的吗？→ 不能凭空编造
- 中间有推进吗？→ 不能只重复开头观点
- 结尾让人想截图吗？→ 没有的话，观众拿什么转发
- 有没有超过 35 字的句子？→ 拆短
- 整体有力量感吗？→ 不能让人觉得"好有道理但跟我没关系"

### 阶段三：生成（Generation）

**目标**：为用户选中的每个文章，从原文提炼核心观点，生成鸡汤文案（长文版+金句版）。

**对每个选中的文章**：

1. **重读原文**，找到核心观点——一句话说清这篇文章想表达什么人生道理
2. **确定提炼角度**——这个观点可以从什么角度写成鸡汤？（成长、放手、自律、心态、人际关系、独处……）
3. **创作长文版**——用开头钩子+中间推进+结尾技巧写成 200-500 字完整鸡汤文。标题自拟，不一定是原标题
4. **创作金句版**——从长文版中提炼 3-10 句独立可转发的金句，情绪递进排列
5. **输出到文件** `{输出目录}/{文章标题}-healing-text.md`（输出目录由用户通过 `--output` 指定，默认为 `.outputs/`）

**文案模板**：

```markdown
# {文案标题（自拟，有观点有态度）}
> 来源：{书籍名} | 作者：{作者} | 栏目：{栏目} | 核心观点：{一句话提炼}

## 长文版

{200-500字完整鸡汤文。开头用观点直给或反差冲击抓住人，中间用递进/举例/对比推进论述，结尾用观点定调/反差回击/行动指令收住。短句为主，每句15-25字。}

## 金句版

- {金句1，15-35字，能独立截图转发}
- {金句2}
- ...（3-10句，情绪递进，三种句式混搭）
```

**生成后，更新数据库**——调用 `mark_selected(story_id)`，再调用 `add_script(story_id, style, path, date)`。style 填写开头使用的模式（"观点直给"或"反差冲击"）。

### 阶段四：记录（Record）

**目标**：确保所有故事（无论选中与否）都已标记在 SQLite 中。

1. 将阶段二中评估过的所有故事，status 不是 `selected` 的，调用 `mark_rejected(story_id)` 统一设为 `rejected`
2. 调用 `get_stats()` 获取最新统计
3. 输出本次处理总结：
   - 📖 处理书籍：X 本
   - 📄 评估故事：Y 篇
   - ✅ 生成脚本：Z 个
   - ❌ 筛选排除：W 篇
   - 📁 输出目录：`{实际使用的输出目录}`

## 评分参考校准

帮助做出一致的评分判断：

**9-10 分（必做）**：
- 有明确的人生观点/态度，能提炼出让人"被点燃"或"被治愈"的鸡汤文，金句密度高
- 例：关于"允许一切发生"的感悟、关于"主动放弃消耗自己的事"的洞察、关于"独处的力量"的表达

**7-8 分（推荐做）**：
- 有可提炼的观点，某方面突出（力量感/治愈感/共鸣感）
- 例：关于成长代价的感悟、关于人际关系的洞察、关于心态调整的建议

**5-6 分（可选做）**：
- 有潜在观点，但不够锋利，需要较多改编才能成为有力鸡汤
- 例：泛泛的"努力很重要"、过于平淡的人生感悟

**3-4 分（通常跳过）**：
- 偏叙事无观点、偏说明/科普、观点太泛无法落地
- 例：纯故事讲述、科普说明文、流水账日记

**1-2 分（直接跳过）**：
- 无可提炼观点、纯数据/技术内容、内容让人绝望无力
- 例：电子元器件科普、悲剧叙事（终点是死亡/绝望）、纯技术文档

## 数据库表结构

SQLite 数据库 `.claude/healing-tracker.db`，通过 `.claude/healing_schema.py` 操作。

```
books
├── id (PK)
├── name (UNIQUE)     — 书名（无扩展名）
├── source_path       — 书籍文件路径
├── source_type       — 读者杂志/书籍/文章合集/其他
├── issue             — 期号或版本
└── processed_date    — YYYY-MM-DD

stories
├── id (PK)
├── book_id (FK → books.id)
├── title             — 文章标题
├── author            — 作者（可空）
├── section           — 栏目（可空）
├── word_count        — 字数（0 表示未知）
├── status            — 'evaluated' | 'selected' | 'rejected'
├── eval_score        — 1-10
├── eval_tags         — JSON 数组字符串
├── eval_best_styles  — JSON 数组字符串
├── eval_notes        — 备注
└── UNIQUE(book_id, title)

scripts
├── id (PK)
├── story_id (FK → stories.id)
├── style             — 叙事类型名
├── path              — 输出文件路径
├── generated_date    — YYYY-MM-DD
└── UNIQUE(story_id, style)

视图：v_stats（全局统计）、v_book_progress（按书进度）、v_story_scripts（故事脚本数）
```

**操作规范**：
- 始终通过 `healing_schema.py` 的函数操作数据库，不手写 SQL
- 每次 `python3 -c` 调用前先 `sys.path.insert(0, '.claude')` 确保导入路径
- 所有文本字段传入前确保是字符串（None/空值用空字符串代替）

## 其他要求

- **所有面向用户的输出用中文**
- **观点不得捏造**——所有观点和洞察必须来自书籍原文，可以提炼、改编、重组，但不能凭空编造
- **段落间用空行分隔**——长文版段落之间留空行，金句版每句独立成段
- **每个生成阶段结束必须报告进度**——让用户知道多少个故事已处理、多少个待处理
- **输出文件命名**：`{输出目录}/{文章标题}-healing-text.md`（输出目录由用户通过 `--output` 指定，默认为 `.outputs/`，目录不存在时自动创建）
- **文案标题自拟**——标题本身就应该是一个有态度的观点或金句，不要直接用原文标题（除非原标题本身就很适合）
