# 电影影评文案生成器

通过搜索全网影评、观众反馈和电影幕后信息，找到电影最打动人的角度，生成有感染力、有观点的剪映图文成片影评文案，并通过本地 SQLite 数据库（`.claude/movie-review-tracker.db`）追踪已处理电影，避免重复生成。

## 核心理念

**电影是体验，影评是共鸣。** 不做机械的"这部电影讲了什么"——而是找到这部电影最击中你的那个瞬间、那句台词、那个眼神，用有感染力的方式告诉观众：**"这部电影值得你花两小时去看。"**

好的影评文案的标准：看完让人觉得 **"我想看这部电影"** 或 **"我也被这个瞬间击中过"** 或 **"原来这个细节是这个意思"** ——不是被剧透了一脸，而是被点燃了好奇心、被戳中了内心、被打开了一个新视角。

### 短视频推荐的底层逻辑

**你在"卖"电影，不是在"写"电影。** 短视频推荐和文字影评是两回事。文字影评可以慢慢品，短视频只有几秒窗口——观众划走只在一念之间。所以：

**1. 姿态是"聊"不是"念"**
文案是口播脚本，不是散文朗读。要像朋友看完电影迫不及待跟你说"哎我跟你说这电影绝了"的口吻，不要像播音员念稿。写完大声读一遍——如果读起来像"作品"，就重写；如果读起来像在跟人聊天，就对了。

**2. 信息节奏层层加码**
不要按时间线平铺"然后呢然后呢"——要让观众每听一句都想知道下一句。最好的结构是：你猜怎么着 → 什么？！→ 不会吧？！→ 还有更绝的？！一个揭示比一个更让人坐不住。剧情可以讲，但每个信息点的释放顺序要按"震撼程度"排，不是按"发生时间"排。

**3. 叙事视角自由切换**
第一人称（"我看的时候..."）、第三人称客观讲述、代入角色视角（"如果你是阿嬷..."）、直接跟观众对话（"你有没有..."）都可以。没有固定公式，选当下最能击中观众的那个视角。一个文案里也可以多种视角混用——只要自然，像聊天一样。

**4. 结尾是行动召唤，不是散文金句**
一个朋友看完电影跟你说什么？"去看吧""带你妈去""别等""信我，这片不一样"。这才是人话。让观众看完文案的第一反应是去搜这部电影——不是觉得"文案写得真好"，而是"我要去看"。

## 使用方法

```
/movie-review <电影名称> [--style 风格] [--output 输出目录] [--notion] [--notion-parent PAGE_ID]
```

- `<电影名称>`：**必填**。电影名称，支持中英文。可以加年份以区分同名电影，如 `《活着》1994`
- `--style`：指定视频风格（深度解读/情感共鸣/快速种草/经典台词/冷门推荐），默认自动匹配最佳风格
- `--output`：指定输出目录，默认为 `studio/`
- `--notion`：上传文案到 Notion（默认不上传，仅生成本地文件）
- `--notion-parent {page_id}`：指定 Notion 父页面 ID，默认读取环境变量 `NOTION_PARENT_PAGE_ID`

示例：
- `/movie-review 肖申克的救赎`
- `/movie-review 流浪地球2 --style 深度解读`
- `/movie-review 千与千寻 --output studio/`
- `/movie-review 霸王别姬 --style 情感共鸣 --notion`

## 数据库工具

所有 SQLite 操作通过 `.claude/movie_review_schema.py` 提供的 Python 函数完成。技能执行过程中，以 `python3 -c` 或 heredoc 方式调用这些函数，无需直接写 SQL。

关键函数：
- `add_movie(title, director, release_year, genre, douban_rating, source, date) → movie_id`
- `get_processed_movies() → list` — 获取所有已处理电影
- `is_movie_processed(title, director) → bool` — 检查是否已处理
- `get_movie_scripts(movie_id) → list` — 获取某电影已有脚本
- `add_script(movie_id, style, title, path, date) → script_id`
- `update_notion_upload(movie_id, style, page_id, page_url)`
- `get_stats() → dict` — 全局统计
- `get_all_movies_with_scripts() → list` — 所有电影及脚本

调用方式示例：
```bash
python3 -c "
import sys; sys.path.insert(0, '.claude')
from movie_review_schema import *
init_db()
stats = get_stats()
print(stats)
"
```

## 核心约束

1. **外部搜索必做**：影评文案的差异化核心就是全网影评搜索。**不可跳过**。必须通过搜索获取豆瓣评分、观众评价、深度影评文章、幕后故事等信息。不做外部调研就无法生成有说服力的影评文案。
2. **每处理一部电影，必须立即通过 `movie_review_schema.py` 写入 SQLite 数据库**。这是防止重复处理的底线机制。
3. **推荐价值 > 剧情复述**：文案的核心不是"这部电影讲了什么故事"，而是"为什么你要花两小时看这部电影"。
4. **控制剧透**：可提及故事设定和情感走向，但不得透露关键反转和结局。用悬念引导观看欲望，而非用剧透满足好奇心。

## 工作流：五阶段（+ 可选 Notion 上传）

### 阶段一：电影发现（Discovery）

**目标**：搜索电影基本信息，了解这部电影"是什么"。

**步骤**：

1. **解析参数**：从 `$ARGUMENTS` 中提取电影名称、`--style`、`--output`、`--notion`、`--notion-parent`

2. **第一轮基本信息搜索**（WebSearch / Tavily，不可跳过）：
   - `{电影名称} 豆瓣` — 获取评分、基本信息、短评
   - `{电影名称} 导演 演员` — 确认主创信息
   - `{电影名称} 上映时间 类型` — 确认年份和类型
   - `{电影名称} 简介` — 了解基本剧情设定（注意不要深度剧透自己）

3. **通过 WebFetch 访问豆瓣页面**获取准确信息（如有豆瓣链接）：
   - 豆瓣评分
   - 评分人数
   - 类型标签
   - 一句话简介
   - 热门短评（前 5 条）

4. **识别电影类型**——根据搜索结果自动判断：
   - `剧情`：以人物命运、社会议题为核心
   - `喜剧`：以幽默搞笑为主要手段
   - `动作/冒险`：以视觉刺激和节奏感取胜
   - `科幻/奇幻`：架空世界观，想象力驱动
   - `悬疑/惊悚`：以悬念和反转为核心
   - `恐怖`：以恐惧感为体验核心
   - `动画`：动画形式，全年龄或成人向
   - `纪录片`：真实事件或人物记录
   - `文艺/独立`：作者表达优先，风格化强
   - `爱情`：以情感关系为核心
   - `战争/历史`：基于历史事件或战争背景
   - `犯罪/黑色`：以犯罪、黑帮、人性阴暗面为主题

5. **检查是否已处理**——调用 `is_movie_processed(title, director)` 检查。如果已处理，列出已有脚本的风格，询问用户是否要生成新风格。

6. **向用户展示电影概况**：
   ```
   🎬 《电影名称》（{上映年份}）
   🎥 导演：{导演}
   🌟 主演：{主要演员，2-4人}
   ⭐ 豆瓣评分：{评分}（{评分人数}人评价）
   📂 类型：{电影类型}
   💡 一句话：{核心看点，一句话}
   ```

**优先级 1**：使用 WebSearch 工具进行搜索
**优先级 2**：WebSearch 返回错误后，降级使用 Tavily Search API：

#### 2.1 第一轮搜索：了解公众认知

**通用搜索（WebSearch / Tavily）**：
- `{电影名称} 影评` — 了解整体评价风向
- `{电影名称} 豆瓣 短评` — 获取代表性观众评价
- `{电影名称} 知乎` — 搜索知乎高赞回答和深度讨论
- `{电影名称} 为什么好看` — 了解推荐理由
- `{电影名称} 最震撼的一幕` — 寻找高光时刻

**微信搜一搜**（如有可用工具，**必执行**）：
- `{电影名称} 影评` — 搜索公众号深度影评文章
- `{电影名称} 推荐` — 搜索推荐类文章

**目标产出**：
- 豆瓣评分及评分分布（几星占比）
- 3-5 条最有代表性的观众短评（正反面都要，一句话短评为佳）
- 这部电影被推荐的理由中最常出现的 3 个关键词
- 公众号深度影评文章 2-3 篇
- 知乎高赞回答 2-3 篇

#### 2.2 第二轮搜索：寻找差异化角度

**通用搜索**：
- `{电影名称} 最打动人的瞬间` — 寻找情感触点
- `{电影名称} 经典台词` — 寻找可引用的台词
- `{电影名称} 细节 隐喻` — 寻找隐藏彩蛋和深层次解读
- `{电影名称} 幕后 拍摄` — 寻找幕后故事和制作细节
- `{电影名称} 导演 创作理念` — 了解创作动机
- `{电影名称} 知乎 分析` — 寻找知乎深度分析长文
- `{电影名称} 知乎 细思极恐` 或 `{电影名称} 知乎 你没看懂` — 寻找知乎独有的挖细节角度

**微信搜一搜（必执行）**：
- `{电影名称} 深度解读` — 寻找有深度的分析视角
- `{电影名称} 隐藏细节` 或 `{电影名称} 你没看懂的` — 寻找差异化解读角度

**获取全文**：搜索到微信文章 URL 后，使用 WebFetch 获取文章正文内容。

**目标产出**：
- 这部电影最打动人的 1 个核心角度（区别于其他影评）
- 3-5 句经典台词或金句（中英对照，如有原版）
- 1-2 个关于拍摄幕后或导演意图的独特素材
- 与同类电影的差异化定位
- 1-2 个容易被忽略但值得关注的细节

#### 2.3 第三轮搜索（可选）：深度挖掘

- `{电影名称} 导演 创作背景` — 了解导演为什么拍这部电影
- `{电影名称} 社会影响` 或 `{电影名称} 为什么成为经典` — 了解影响力
- `{电影名称} 类似电影` — 为差异化对比和"如果喜欢XX"推荐找素材
- `{电影名称} 获奖` — 了解奖项荣誉，增加说服力
- `{电影名称} 知乎 如何评价` — 知乎标志性评价体，寻找多元观点碰撞

#### 2.4 搜索结果存档

**⚠️ 搜索结果必须存档**。在输出目录下创建 `attachments/` 目录，保存搜索索引和文章内容。

**搜索索引文件**（`attachments/搜索索引.md`）记录所有搜索关键词、搜索工具、结果数、关键发现。

**文章内容文件**（`attachments/文章{N}-{标题}.md`）保存搜索到的每篇文章全文，供后续事实核查和文案生成回溯引用。

### 阶段三：文案生成（Generation）

**目标**：基于电影信息 + 全网影评素材，生成剪映图文成片影评文案。

#### 3.1 自动匹配最佳风格

如果用户未指定 `--style`，根据电影类型和调研发现自动选择最佳风格：

| 电影类型 | 默认风格 | 理由 |
|----------|----------|------|
| 剧情/文艺/独立 | 深度解读 | 有人物深度和社会议题，适合深入剖析 |
| 爱情/家庭/成长 | 情感共鸣 | 有情感弧光，适合共情切入 |
| 喜剧/动作/冒险 | 快速种草 | 娱乐性强，适合直击爽点 |
| 科幻/奇幻/悬疑 | 深度解读 | 有世界观构造和隐藏细节，适合"揭秘"角度 |
| 恐怖/惊悚 | 快速种草 | 体验型观影，适合感官描述驱动 |
| 动画/合家欢 | 情感共鸣 | 往往有温暖内核，适合治愈角度 |
| 纪录片/战争/历史 | 深度解读 | 有真实分量，适合知识增量角度 |
| 犯罪/黑色 | 经典台词 | 往往有犀利对白和名场面 |
| 冷门/小众/独立 | 冷门推荐 | 天然差异化，好奇心驱动 |

**风格选择指南**（如果调研发现更强角度，可覆盖默认匹配）：
- 外部影评中情感共鸣评价特别多 → 优先情感共鸣
- 电影中有大量经典台词和名场面 → 优先经典台词
- 电影核心主题极其反常识或有深度 → 优先深度解读
- 电影娱乐性和观感体验极强 → 优先快速种草
- 电影评分高但知名度低 → 优先冷门推荐

#### 3.2 影评文案创作技巧

##### 五种风格指南

**风格一：深度解读**
- **开场钩子**：一个颠覆大众认知的解读角度，或一个被99%观众忽略的关键细节
- **中间推进**：层层递进——先讲表面的故事 → 再讲隐藏的暗线/隐喻 → 最后讲导演真正想表达的东西 → 这个主题如何映照现实
- **结尾**：一个让人想立刻重看一遍验证的悬念或细节——不是"所以这部电影告诉了我们..."，而是"下次你再看的时候，注意看这个镜头..."
- **适合电影**：《肖申克的救赎》《寄生虫》《盗梦空间》《少年派的奇幻漂流》

**风格二：情感共鸣**
- **开场钩子**：一个让人瞬间共情的场景描述，或用"你有没有..."直接拉观众入戏
- **中间推进**：先讲角色的处境（让观众代入自己或身边人）→ 再讲关键转折（情绪升温）→ 最后讲这个故事的普世情感意义
- **结尾**：不是金句，是行动召唤——"带包纸巾""带妈妈去看""别等到来不及"。让人沉默三秒，然后打开购票App
- **适合电影**：《忠犬八公》《你好李焕英》《泰坦尼克号》《美丽人生》

**风格三：快速种草**
- **开场钩子**：直击观影动机——一句话让观众觉得"这就是我要看的"
- **中间推进**：观影期待放大 → 这部电影能带来什么体验（给感受不给剧透）→ 看完后的后劲有多大
- **结尾**：一个具体的时间/场景/观影方式建议——"今晚关灯一个人看""去电影院，大银幕完全不一样""约朋友一起刷，出来还能聊一晚上"
- **适合电影**：《疾速追杀》系列、《头号玩家》、《让子弹飞》、《疯狂的石头》

**风格四：经典台词**
- **开场钩子**：直接抛出电影中最炸的一句台词，让观众瞬间进入电影氛围
- **中间推进**：第一句台词 + 出现的场景（为什么炸）→ 第二句台词 + 它道出了什么真相 → 第三句台词 + 三句放在一起揭示的电影内核
- **结尾**：用一句最有力但最安静的台词收尾，留白。让观众自己去搜、去听完整的
- **适合电影**：《让子弹飞》《无间道》《大话西游》《教父》

**风格五：冷门推荐**
- **开场钩子**："冷门但炸裂"的强烈反差——评分高/口碑好但看过的人不多
- **中间推进**：为什么冷门（宣传少/题材小众/年代久远/无大牌）→ 为什么炸裂（具体片段或细节证明）→ 看完的感受（"捡到宝了"）
- **结尾**：制造稀缺感和社群感——"看过的人不多，但评分不会骗人""趁还没下架，赶紧"
- **适合电影**：评分8.5+但标记人数少于5万的冷门佳片

##### 通用写作规范

0. **⚠️ 内容安全红线（强制遵守，违反即重写）**：
   - ❌ 禁止对现实国家/政府/政策进行批判
   - ❌ 禁止制造群体对立（地域、性别、阶层等）
   - ❌ 禁止对现实中的品牌/企业进行负面评价
   - ❌ 禁止过度负面情绪宣泄和煽动
   - ✅ 替代方案：聚焦电影本身的主题探讨，用"电影让我们看到..."的视角

1. **情绪线不能断（最重要，优先级高于一切）**：
   - 抖音观众划走只有一个原因：情绪断了。情绪一断，一切白费
   - **每种风格都有不同的情绪曲线（见上方各风格描述）**
   - **⚠️ 反塌房铁律**：视频后半段禁止变成以下任何一种——
     - ❌ 片单罗列（"除此之外还有 XX、YY、ZZ 也值得看"）
     - ❌ 导演生平报告（"导演1968年出生于..."）
     - ❌ 教科书式总结（"这部电影一共分为三幕..."）
     - ❌ 观影方法论说教（"我们应该这样看电影..."）
     - ❌ 强行升华（"所以我们要珍惜当下，活在当下"）
     - ❌ 散文式金句收尾（"做人呐，要有情有义——这七个字，是这个时代最稀缺的东西"）
   - **后半段该怎么写**：用一个更大的反转、一个更深的触动、一句更炸的台词，或者一个让观众忍不住立刻打开观影App的行动召唤来收尾。记住：观众要的不是总结，是冲动

2. **推荐价值是核心**：观众看完不需要知道电影的全部剧情——只需要知道"这部电影能给我什么体验"

3. **结构适配剪映分镜**：
   - 用空行分段，每段对应一个镜头画面
   - 每段 1-2 句话，15-40 字
   - 总字数 600-750 字（对应 2-3 分钟视频，按语速 250 字/分钟）
   - **段落数 10-12 段**，每一帧都要有存在的理由
   - **⚠️ 无填充段规则**：删掉该段后视频仍然通顺 = 这段是废话，必须删
   - 首段至少 15 字以上

4. **开头 3 秒钩子**：前 3 秒必须抛出一个**具体到能立刻制造好奇**的前提，并以一个**没回答的问题**收尾，逼观众看下去。
   - ✅ 好开头 = 具体的人 + 不可能的事 + 一个问号。例："一个阿嬷等丈夫的信等了几十年。孙子出国一查才知道——丈夫六十年前就死了。可那些信，在他死后又整整寄了十八年。谁在冒充一个死人？"
   - ❌ 坏开头 = 泛泛的情绪反问（"你有没有在电影院哭过？""你想过吗？"）——观众听完还是不知道你要讲什么，直接划走
   - 绝不能是"今天推荐一部电影"，也绝不能是"你有没有过XX"这种万能反问

5. **口语化表达（聊，不是念）**：短句为主，像一个爱电影的朋友在跟你聊天，不是朗读影评文章。**写完必须大声读一遍**——凡是读起来像播音、像散文、像有声小说的句子，一律改成大白话。检查标准：把这句文案发微信给朋友，对方会不会觉得你在正常聊天？会 → 过。不会 → 改

6. **具象化改写（剪映适配核心）**：

| 类型 | 问题 | 改写方向 |
|------|------|---------|
| 抽象概念（"自由""命运""救赎"） | 剪映无法匹配画面 | 替换为具体的场景/动作/物品（"牢房墙上的海报""雨中的拥抱"） |
| 心理描述（"他意识到..."） | 无法拍摄 | 替换为外化的动作/表情/镜头语言（"他站在雨中，一动不动"） |
| 影评术语（"长镜头调度""蒙太奇"） | 观众难以理解 | 改成感受描述，如"那一段一镜到底"→"镜头跟着他，一刻没离开" |
| 数据/票房/评分 | 需要视觉锚点 | 加具体场景，如"上映三天，全国影院排片翻了五倍" |

7. **结尾（在情绪最高点落地，别狗尾续貂）**：高潮镜头（一句台词/一个眼神）过后，最多再用一个会回响的画面、或一句让片名"咔哒"扣上的话收尾，然后停。
   - ❌ 两大忌：
     - (a) 突然甩票房/评分/成本数据（"豆瓣9.3，成本一千四百万，票房十七亿"）——这是影评报告不是口播，瞬间打断情绪
     - (b) 命令式说教（"带包纸巾""带你妈去""别等""信我"）——像在教训观众，让人反感
   - ✅ 好结尾：落在一个具体画面 + 点题片名，让观众自己生出"我要去看"的冲动。例："那封信，从来不是丈夫写的。可那份情，一分不假。这部电影，叫《给阿嬷的情书》。"
   - 结尾检验：读完后第一反应是觉得"写得好"还是"我要去看"？前者说明你在炫技，后者才对

8. **反cliché规则（严禁使用）**：
   - ❌ "今天要给大家推荐一部好电影" / "这部电影改变了我的人生" / "看完我才明白"
   - ❌ "毫不夸张地说" / "强烈推荐" / "相信我" / "你怎么看评论区告诉我"
   - ❌ "演技炸裂" / "全程无尿点" / "年度最佳" / "封神之作"
   - ✅ 替代方案：用一个具体的镜头/一句台词/一个感受来推荐，让观众自己得出结论

##### 差异化策略（至少用一种，优先组合使用）

- **反直觉切入**：从大众对这部电影的常见误解入手，用深度解读打破误解。如："都以为这是部越狱片，其实它讲的是希望如何在一个人的骨头里生根"
- **普通人关联**：不讲电影的艺术成就，讲这部电影如何映照普通人的真实困境。如："肖申克的监狱不一定是石墙，也可能是你不敢辞的工作、你不敢分的手"
- **隐藏的暗线**：电影中最容易被忽略的配角/细节/道具，却藏着导演最想说的话
- **跨时空对比**：这部电影上映时被低估/被误解，X 年后的今天再看，才发现它预言了什么/道出了什么
- **第一人称叙事**：用"我第一次看的时候..."的视角，带出个人观影体验的变化，而不是上帝视角评价
- **放大最毒的细节**：电影中最触动你的那一个眼神、一句台词、一个镜头，放大为全片核心线索

##### 视频简介（发布文案，必出）

每条文案除口播台词外，**必须**再产出一段"视频简介"——发布时填在视频下方的文字栏，让刷到视频的人**不点开也能三秒看懂**。

**要求**：
- **简短**：2-4 句，80-150 字
- **三要素必含**：
  1. 点明这是一部电影（让算法/搜索识别内容类型）
  2. 写清**片名**（命中搜索；遇同名片加年份/导演区分）
  3. 这条视频**最想表达的那一个点**——从口播里提炼出最有态度的一句，**不是复述剧情**
- **语气**：和口播一致，口语、有态度；末尾可加 1-3 个话题标签（`#片名` `#类型`）
- **禁止**：剧透关键反转和结局；把剧情再讲一遍；广告话术（"强烈推荐""年度必看"）

**示例**（《给阿嬷的情书》）：
> 一个阿嬷等丈夫的信等了六十年，才知道丈夫早就不在了——可那些信，在他死后又整整寄了十八年。字是假的，情是真的。🎬 电影《给阿嬷的情书》（2026），豆瓣9.3。#给阿嬷的情书 #电影推荐 #潮汕

##### 素材搜索列说明

素材搜索列为**辅助参考**，剪映图文成片会自动根据台词匹配画面。只有当某些段落的自动匹配效果不理想时，才需要用到这些搜索词去找替换素材。

| 来源 | 适用场景 | 搜索词构建要点 |

#### 3.4 文案自检（11 项质量检查，强制执行）

**⚠️ 这是一个强制执行的质量关卡，不可跳过。每个文案文件必须逐项通过以下 11 条检查。**

| # | 检查项 | 检查方法 | 不通过则 |
|---|--------|----------|----------|
| 0 | **⚠️ 内容安全** | 是否存在对现实国家/政府/政策的批判？制造群体对立？对品牌/企业的负面评价？过度负面情绪？ | 删掉或用正向角度重写 |
| 1 | **段落数** | 台词纯文本段落数是否在 10-12 段范围内？ | 多则合并相似场景，少则拆分过长的段落 |
| 2 | **总字数** | 台词纯文本总字数是否在 600-750 字？ | 超出则删减填充段，不足则补充关键细节 |
| 3 | **反塌房** | 后半段（第 7 段起）是否存在片单罗列/导演生平报告/教科书总结/观影方法论说教/强行升华/散文式金句收尾？ | 删掉，用更大的反转、更深的触动或一句"人话"行动召唤替换 |
| 4 | **反cliché** | 是否包含禁用话术："今天要给大家推荐" / "演技炸裂" / "全程无尿点" / "年度最佳" / "封神之作" / "毫不夸张地说" / "强烈推荐" / "相信我" / "你怎么看评论区告诉我"？ | 替换为具体场景/细节/感受 |
| 5 | **正向情绪曲线** | 逐段读台词，情绪是否持续升级？升级方向是否正向（被点燃/被治愈/被触动/被启发/被震撼）？是否有断点或转向负面？ | 删掉断点段或负面情绪段 |
| 6 | **3 秒钩子** | 第 1 段是否是一个具体到能立刻制造好奇的前提 + 一个没回答的问题？还是"你有没有过XX"这种万能反问（观众听完还不知道你要讲什么）？ | 重写第 1 段：具体的人 + 不可能的事 + 问号 |
| 7 | **差异化** | 这个推荐角度跟其他影视博主的推荐角度是否不同？搜索确认是否已有大量同类内容？ | 换角度重写或放弃 |
| 8 | **事实核查** | frontmatter 中 verified_claims 和 unverified_claims 是否完整？豆瓣评分、获奖信息是否准确？ | 补充核查结果，模糊化不确定的具体数据 |
| 9 | **具象化** | 是否存在抽象概念/影评术语/文学化表达未被替换为具体场景或感受描述？ | 替换为具体的镜头/场景/动作/感受 |
| 10 | **推荐价值验证** | 读完台词全文，观众最可能的反应是什么？是想去搜这部电影/买票/告诉朋友"有部片你一定要看"？还是觉得"文案写得不错"但不想动？ | 把结尾从"金句总结"改成"行动召唤"，让观众觉得"我要去看"不是"写得好" |
| 12 | **结尾不狗尾续貂** | 高潮过后是不是突然甩票房/评分/成本数据？或用"带你妈去""别等""信我"命令观众？结尾有没有落在情绪最高点之后的一个回响画面/一句让片名扣上的话？ | 删掉数据罗列和命令式说教，用一个具体画面 + 点题片名收住 |
| 11 | **念 vs 聊** | 把台词纯文本大声读一遍。读起来像播音员念稿/散文朗诵/有声小说吗？还是像朋友看完电影在跟你聊天？把文案发微信给朋友，对方会觉得你在正常安利一部电影吗？ | 把"书面语"改"口头语"，把"描述"改"对话"，长句拆短，"的"字删掉一半 |

## 错误处理

| 场景 | 处理 |
|------|------|
| 电影名称模糊（多部同名） | 列出候选电影（含年份和导演），请用户确认 |
| 搜索不到电影信息 | 提示检查电影名称拼写，建议加年份重试 |
| 豆瓣页面无法访问 | 降级使用搜索结果中的评分和评价信息 |
| 电影已处理同风格 | 提示已有同风格脚本，询问是否覆盖 |
| WebSearch 不可用 | **⚠️ 必须降级到 Tavily API（curl 调用），不可跳过** |
| 所有搜索工具都失败 | 提示"无法进行外部调研，影评文案质量无法保证"，询问是否基于基本信息生成 |
| 搜索无影评结果 | 基于基本信息生成，标注"外部影评参考有限" |
| 事实核查无法验证 | 记录到 unverified_claims，具体数据模糊化 |
| 后半段塌房 | **⚠️ 删掉报告/说教，用反转/台词/触动替换** |
| 内容安全违规 | **⚠️ 必须重写，用正向角度切入** |
| Notion 上传失败 | 本地文件不受影响，记录失败原因 |

```
├── source            — 信息来源（如豆瓣URL）
├── processed_date    — YYYY-MM-DD
└── UNIQUE(title, director, release_year)

scripts
├── id (PK)
├── movie_id (FK → movies.id)
├── style             — 深度解读/情感共鸣/快速种草/经典台词/冷门推荐
├── title             — 视频标题
├── path              — 文案文件路径
├── generated_date    — YYYY-MM-DD
├── notion_page_id    — Notion 页面 ID（可空）
├── notion_page_url   — Notion 页面 URL（可空）
├── notion_uploaded   — 0 或 1
└── UNIQUE(movie_id, style)

视图：v_stats（全局统计）、v_movie_scripts（电影脚本列表）
```

**操作规范**：
- 始终通过 `movie_review_schema.py` 的函数操作数据库，不手写 SQL
- 每次 `python3 -c` 调用前先 `sys.path.insert(0, '.claude')`

## 其他要求

- **所有面向用户的输出用中文**
- **影评内容不得捏造**——所有观点和引用必须来自搜索结果验证的外部影评，或从公开信息可确认的剧情细节
- **控制剧透量**——只透露必要的剧情设定来吸引观众，关键反转和结局必须留白
- **段落间用空行分隔**——台词纯文本每段之间留空行
- **每个阶段结束必须报告进度**——让用户知道搜索了多少篇影评、找到了什么关键信息
- **输出文件命名**：`{output_dir}/{YYYY-MM-DD}/{HHMM}/选题1-{标题缩写}.md`
- **视频标题自拟**——标题本身就应该有观点有态度，不要用"XX影评""XX电影推荐"这种泛标题
